STEP 1
MINUTES
▶
get_projected_stat_line() — Minutes Projection
min_weights = [0.40, 0.20, 0.20, 0.10, 0.10] // 5-game recency weighting: most recent game = 40%, then 20/20/10/10
recent_min_avg = weighted average of last 5 games MIN

If player salary ≥ $18 OR recent_min_avg ≥ 28: // Starter tier   proj_min = max(recent_min_avg, 28.0) // Floor 28 minutes for starters
Else: proj_min = recent_min_avg // Role players: use their actual recent average
↓
STEP 2
STAT LINE
▶
get_projected_stat_line() — Per-Stat Projections

Each stat is projected using last-3-game per-minute efficiency, scaled to tonight's projected minutes.

per_min_rate = SUM(last 3 games stat) / SUM(last 3 games MIN) // How many points/rebounds/etc. per minute over last 3 games

PROJ_PTS = per_min_rate(PTS) × proj_min
PROJ_REB = per_min_rate(REB) × proj_min
PROJ_AST = per_min_rate(AST) × proj_min
PROJ_STL = per_min_rate(STL) × proj_min
PROJ_BLK = per_min_rate(BLK) × proj_min
PROJ_TOV = per_min_rate(TOV) × proj_min
PROJ_3PM = per_min_rate(FG3M) × proj_min
⚠ If Back-to-Back game detected → 7% reduction applied to final projected score
↓
STEP 3
RAW FP
▶
Yahoo Fantasy Points Formula
tonight_FP =
  PROJ_PTS × 1.0 + PROJ_REB × 1.2 + PROJ_AST × 1.5
  + PROJ_STL × 3.0 + PROJ_BLK × 3.0 + PROJ_TOV × −1.0
↓
STEP 4
PROJ SCORE
▶
Projected Fantasy Score — Weighted 30/50/20 Blend
WEIGHT INPUT SOURCE WHY
30% tonight_FP Last-3-game efficiency × proj_min Tonight's per-minute rate (most responsive)
50% seven_day_avg Actual FP from games in last 7 days Largest weight — recent form, not distorted by one game
20% FPPG (CSV) Yahoo's season fantasy points per game Season baseline — anchors against streaks/slumps
Projected_Fantasy_Score = (tonight_FP × 0.30) + (seven_day_avg × 0.50) + (FPPG × 0.20) // B2B penalty: × 0.93 if playing back-to-back
↓

INJURY STATUS REFERENCE
INJURY STATUS — WHAT EACH VALUE MEANS IN THE LINEUP ENGINE

Any of these statuses block the player from appearing in auto-generated lineups or manual slot adds. Case-insensitive — "out", "OUT", "Out", "O" all blocked.

OUT — not playing
O — shorthand OUT
INJ — injured
INJURED — injured
IR — injured reserve
INACTIVE — inactive list
SUSPENDED — suspended
DND — did not dress
GTD — game-time decision (ALLOWED)
HEALTHY — playing (ALLOWED)
(blank) — assumed healthy (ALLOWED)

3 LINEUP OPTIMIZATION MODES
MEDIAN MODE (CASH)
Maximizes VALUE_SCORE (salary efficiency). Best for Head-to-Head and 50/50 contests. Selects consistent, floor-safe players.
Sort by: VALUE_SCORE = Projected_FP ÷ Salary
CEILING MODE (GPP)
Targets high-variance, boom-bust players. Optimizes for maximum ceiling FP. Best for tournaments where you need to win big.
Sort by: FP_MAX (ceiling) with 1.60× multiplier cap
MAXIMUM MODE (HAIL MARY)
Highest-usage, highest-floor studs only. USG ≥ 20% AND 7-day avg ≥ 15 FP required. Uses Projected_Fantasy_Score as optimizer metric.
Filter: usage_pct ≥ 20% + 7-day avg ≥ 15 FP → max Projected_FP
NS POWER BOOST
NeuralStitch premium mode. Auto-fills 8 slots using powerBoostIndex (multi-signal blend). Salary cap enforced. No injured players.
Sort by: powerBoostIndex → greedy backtrack solver
